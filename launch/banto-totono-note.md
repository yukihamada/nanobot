# 声だけで請求書が完成する時代へ -- 建設業のDXを本気で考えた

こんにちは。建設業向けの業務効率化ツール「BANTO」を開発しているエンジニアです。

今日は、なぜこのプロダクトを作ったのか、そしてどんな課題を解決しようとしているのかについて書いてみます。

## 現場から聞こえてきた「本音」

ある日、建設業を営む知人にこんなことを言われました。

> 「夕方、疲れて事務所に戻ってからExcelで請求書を作るのが一番キツい」

これは一人親方や中小の建設会社で働く人たちにとって、驚くほど共通した悩みでした。

現場仕事を朝から晩までこなして、ヘトヘトの状態でPCの前に座る。Excelのテンプレートを開いて、今日の作業内容を思い出しながら一行ずつ入力していく。単価を確認して、数量を入れて、消費税を計算して......。

正直、この作業に「知的労働」と呼べる要素はほとんどありません。頭の中にある情報を、決まったフォーマットに転記しているだけ。それなのに、毎回30分から1時間かかる。月末になれば何十枚もの請求書と格闘することになります。

「スマホで声を吹き込むだけで請求書ができたらいいのに」

その一言が、BANTOの開発のきっかけでした。

## じゃあ、声で作れるようにしよう

BANTOの使い方はシンプルです。

**ステップ1: スマホに向かって話す**

たとえばこんな感じです。

> 「今日、山田工務店さんの現場で、大工工事を8時間やりました。単価は1万8千円で。あと材料費で2x4材を20本、1本550円で使いました」

**ステップ2: AIが内容を解析**

音声をテキストに変換し、AIが請求書の構造に自動で整理します。取引先名、作業内容、数量、単価、材料費、消費税率......すべて自動で振り分けます。

**ステップ3: PDFで請求書が完成**

確認画面で内容をチェックして、OKならそのままPDF出力。取引先へのメール送信もワンタップです。

これだけ。現場から帰る車の中で、請求書が完成します。

## 「ちゃんと合ってるの？」という不安への答え

音声入力と聞いて、多くの方が気にするのは精度です。「声で入力して、金額が間違ってたらシャレにならない」と。

もっともな心配です。だからBANTOでは、計算ロジックに徹底的にテストをかけています。

- 請求書の金額計算: 37パターンのテスト済み
- 入力値のバリデーション: 41パターンのテスト済み
- 消費税計算（10%/8%/軽減税率）: すべて端数処理まで検証済み

合計133のテストケースで、計算の正確性を担保しています。AIが作った請求書は、必ず確認画面を挟むので、送信前に内容を目視チェックできます。「AIに丸投げ」ではなく「AIが下書きを作って、人間が最終確認する」という設計です。

## 請求書だけじゃない

BANTOで作れる書類は請求書だけではありません。

- **見積書** -- 現場の下見をしながら、その場で見積もりを作成
- **発注書** -- 材料の発注もスマホから
- **完了報告書** -- 工事完了の報告書も音声入力で

建設業の書類仕事をまるごとカバーします。

## 取引先とのやり取りもスムーズに

作成した請求書はPDFとして出力できるだけでなく、メールでの送信にも対応しています。取引先ごとの履歴も残るので、「先月の請求書、どこやったっけ？」という場面もなくなります。

さらに、入金管理機能もあるので、「この請求書、もう入金されたっけ？」という確認作業も一元化できます。

## なぜ建設業に特化したのか

世の中にはfreeeやMFクラウドなど、優れた請求書サービスがたくさんあります。でも、建設業の現場で使うには、ちょっとした壁がありました。

**1. 工賃と材料費の分離**

建設業の請求書では「人工（にんく）」と「材料費」を分けて記載するのが一般的です。汎用的な請求書ツールだと、この構造がフィットしないことがあります。

**2. 現場での入力のしやすさ**

手袋をしたまま、汗だくの状態で、小さなスマホ画面にフォーム入力するのは現実的ではありません。音声入力なら、手がふさがっていても使えます。

**3. ITリテラシーの前提**

建設業の方々が全員PCに詳しいわけではありません。「話すだけ」というインターフェースは、デジタル機器に不慣れな方でも直感的に使えます。

## スマホアプリとしても使える

BANTOはWebアプリですが、iOSとAndroidのネイティブアプリとしてもインストール可能です。Capacitorを使ったハイブリッドアプリなので、カメラで領収書を撮影する機能なども使えます。

ホーム画面にアイコンを追加すれば、普通のアプリと同じ感覚で使えます。

## 技術的な裏側（エンジニア向け）

少しだけ技術的な話を。BANTOは以下の技術スタックで構築しています。

- React 18 + TypeScript + Vite（高速な開発体験）
- shadcn/ui + Tailwind CSS（モダンなUI）
- Supabase（PostgreSQL + 認証 + ストレージ）
- Playwright + Vitest（E2Eテスト + ユニットテスト）

サーバーレスアーキテクチャなので、運用コストを抑えながらスケーラブルな構成になっています。

## まずは試してみてください

BANTOは現在、無料で使えるプランを用意しています。

「声で請求書を作る」という体験は、一度やってみると、もうExcel入力には戻れなくなります。

特に一人親方の方や、少人数で回している建設会社の方に、ぜひ試していただきたい。

**BANTO -- 声だけで請求書完成**
https://banto.jp

現場の帰り道が、事務作業の時間に変わります。

# 月額SaaS費用を1/3に。10のツールを1つにまとめた話

こんにちは。統合ビジネスプラットフォーム「Totonos」を開発しています。

今回は、中小企業やスタートアップが避けて通れない「SaaS費用問題」について、私たちなりの答えを書いてみます。

## 毎月のSaaS請求書、合計したことありますか？

ちょっと計算してみてください。

- freee（会計）: 月額3,980円
- SmartHR（人事労務）: 月額数万円
- Salesforce（CRM）: 1ユーザー月額3,000円〜
- Notion（ドキュメント）: 1ユーザー月額1,650円
- クラウドサイン（電子契約）: 月額11,000円
- 楽楽精算（経費精算）: 月額30,000円〜
- Asana（プロジェクト管理）: 1ユーザー月額1,475円

10人のチームで、これらを全部使うと月額20万円を軽く超えます。年間で240万円以上。

スタートアップの初期フェーズや、中小企業の管理部門にとって、この金額はかなりの負担です。しかも、ツールが分散していることで生まれる「運用コスト」もバカになりません。

## 分散の本当のコスト

金額だけの問題ではないんです。

**データがバラバラ**。CRMで管理している取引先情報と、会計ソフトの取引先マスタが一致しない。請求書を発行するたびに、CRMの画面と会計ソフトの画面を行ったり来たり。

**アカウント管理が大変**。社員が入社するたびに7つのサービスにアカウントを作り、退職するたびに7つのサービスからアカウントを削除する。1人分の設定に半日かかることも。

**学習コストが高い**。新しいメンバーが入るたびに、7つのツールの使い方を覚えてもらう必要がある。研修資料も7つ分。

こうした「見えないコスト」を含めると、実際の負担はSaaS利用料の2〜3倍になるという試算もあります。

## Totonos という答え

Totonos は、これらの機能を1つのプラットフォームに統合しました。

| 機能 | 置き換わるサービス |
|------|------------------|
| CRM（リード・商談・取引先管理） | Salesforce |
| 請求書・見積書作成 | freee、MFクラウド |
| 電子契約・契約書管理 | クラウドサイン |
| 従業員管理・勤怠・給与 | SmartHR、HRMOS |
| 経費精算・領収書OCR | 楽楽精算 |
| 仕訳帳・勘定科目・財務レポート | freee、MFクラウド |
| ナレッジベース・Wiki | Notion |
| タスク・プロジェクト管理 | Asana |
| AIアシスタント | ChatGPTなど |
| LMS（学習管理） | - |
| 会員管理 | - |

1つのログインで、すべての業務が完結します。

## 「統合」のメリットは想像以上

正直に言うと、最初は「各機能が中途半端になるんじゃないか」と心配しました。でも実際に使ってみると、統合によるメリットが個別最適を上回るケースがほとんどでした。

**例1: 請求書の発行**

CRMで商談が成約したら、そのままワンクリックで請求書を発行。取引先情報は自動で引き継がれるので、再入力は不要。発行した請求書は会計モジュールに自動で仕訳される。

**例2: 経費精算**

スマホで領収書を撮影すると、OCRで内容を読み取り、経費申請が自動作成される。承認されたら会計の仕訳に直接反映。

**例3: 社員のオンボーディング**

1つのアカウントを作るだけで、CRMもWikiもプロジェクト管理もすべてアクセス可能に。権限設定も1箇所で完結。

データが1つの場所にあるということは、こういう連携が「設定なし」で動くということです。

## 業種別テンプレートで、すぐに使える

Totonos は汎用的なプラットフォームですが、業種ごとに最適な初期設定をテンプレートとして用意しています。

- **小売**: アパレル、家具、ペットショップ
- **サービス**: 飲食店、美容室、フィットネス、ホテル
- **専門サービス**: 法律事務所、会計事務所、コンサルティング
- **医療・福祉**: クリニック、歯科、薬局、介護施設
- **建設・不動産**: 建設会社、不動産仲介、リフォーム
- **IT**: SaaS、Web制作、システム開発
- **教育**: 学習塾、語学学校

初回ログイン時に業種を選ぶだけで、メニュー構成や入力項目がその業種に最適化されます。

## AIアシスタントが業務をサポート

各機能にAIアシスタントが組み込まれています。

- **売上予測**: 過去データから売上をAIが予測
- **リードスコアリング**: 見込み客の優先度をAIが判定
- **領収書OCR**: 撮影した領収書の内容をAIが自動入力
- **チャットボット**: 業務に関する質問に即座に回答

AIの利用にはクレジット制を採用しているので、使った分だけのコスト。固定費が膨らむ心配はありません。

## セキュリティも妥協しない

中小企業だからといって、セキュリティを甘く見るわけにはいきません。

- **Row Level Security（RLS）**: データベースレベルでのアクセス制御。他社のデータは絶対に見えない設計
- **二要素認証（2FA）**: 全ユーザーに対応
- **監査ログ**: 誰がいつ何をしたか、すべて記録

## オープンソースで、ベンダーロックインなし

Totonos はMITライセンスのオープンソースです。

つまり、もし将来的に「自社のサーバーで動かしたい」と思ったら、それも可能です。Vercel、Cloudflare Pages、Netlify、AWS、Firebase、Fly.io、Docker......好きな環境にデプロイできます。

「このSaaS、もしサービスが終了したらデータどうなるの？」という不安からも解放されます。

## 技術スタック（エンジニア向け）

- React 18 + TypeScript + Vite
- shadcn/ui + Tailwind CSS
- Supabase（PostgreSQL + Auth + Edge Functions）
- Anthropic Claude API（AIアシスタント）
- Stripe（決済）
- Tiptap（リッチテキストエディタ）
- i18next（多言語対応）

557以上のテストケース（ユニット + インテグレーション + E2E）で品質を担保しています。

## こんな方に使ってほしい

- スタートアップの管理部門で、SaaS費用を抑えたい方
- 中小企業で、ツールの分散に疲れている方
- 新しい会社を立ち上げて、最初からすべてを1つにまとめたい方
- ITに詳しくないけど、業務をデジタル化したい方

Totonos は無料プランから始められます。

SaaS費用を1/3にするという話は、大げさに聞こえるかもしれません。でも、7つのツールの月額費用を合計して、Totonos 1つの費用と比較してみてください。数字が答えを出してくれます。

https://totonos.jp

あなたのビジネスの「散らばった業務」を、1つにまとめます。
