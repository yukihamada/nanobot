Key Differentiators

🎤 Voice-First

The only open-source AI agent framework with native STT + TTS + push-to-talk UI. Talk to AI like you're on a phone call. Auto-TTS reads responses aloud when voice input is used.

web/index.html — Web Speech API + /api/v1/speech/synthesize

📱 14+ Channels

LINE, Telegram, WhatsApp, Discord, Slack, Teams, Facebook, Zalo, Feishu, Google Chat, and more. Cross-channel conversation sync via /link command and QR code. OpenClaw has 13+ channels, Picoclaw has 2, ChatGPT has 4.

crates/nanobot-core/src/channel/ — line.rs, telegram.rs, facebook.rs, ...

⚡ Fastest Runtime

Rust on Lambda ARM64 = sub-50ms cold start, <2s response. Picoclaw (Go) starts in ~100ms, OpenClaw (Node.js) in ~1s. No Python overhead. 4.6MB binary, stripped with LTO.

crates/nanobot-lambda/ — Compiled Rust on AWS Lambda

🔄 Auto Failover

Primary model fails? Automatically retries with gpt-4o-mini, then gemini-2.0-flash-lite. Zero errors shown to user. Load balancing across multiple API keys. No other framework offers this.

crates/nanobot-core/src/service/http.rs — handle_chat() failover logic

🧠 Long-Term Memory

2-layer memory system: daily conversation logs + long-term facts. Auto-consolidated every 10 entries via cheap LLM. Picoclaw uses SOUL.md files, ChatGPT has cross-conversation memory, but neither auto-consolidates.

crates/nanobot-core/src/service/http.rs — read_memory_context(), append_daily_memory()

☁️ Serverless Scale

Zero-to-infinite scale on AWS Lambda. No VPS to manage. DynamoDB single-table design. OpenClaw and Picoclaw require self-hosting. ChatGPT is cloud-only but not self-hostable.

infra/template.yaml — SAM template for Lambda + DynamoDB + API Gateway
